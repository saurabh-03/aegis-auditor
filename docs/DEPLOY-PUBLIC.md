# Launching Aegis for everyone (public deployment)

This guide takes Aegis from "runs on my laptop" to "anyone on the internet can use it."

> **Read the safety section first.** A public security scanner has real abuse and
> legal considerations. Aegis ships hardened by default, but you own the instance.

---

## What "public" needs (the honest checklist)

| Requirement | Status | Notes |
|---|---|---|
| Long-running server (not serverless) | you provide | WebSockets + workers + raw sockets can't run on Vercel/Lambda |
| PostgreSQL + Redis | provisioned by the blueprint | persistence + job queue |
| A strong `JWT_SECRET` | auto-generated by Render | never commit it |
| HTTPS | provided by the host | Render/Railway/Fly give TLS automatically |
| SSRF protection | **built-in, on by default** | blocks scans of private/internal/metadata IPs |
| Rate limiting | built-in | tightened to 10/min on the blueprint |
| A domain (optional) | you provide | otherwise you get `*.onrender.com` |

Only two things require **you**: a hosting account, and (optionally) a domain.
Everything else is in the repo.

---

## Recommended path — Render (git-connected, free tier, one file)

Your repo already contains [`render.yaml`](../render.yaml), which defines the whole
stack (Postgres + Redis + API + worker + web dashboard).

1. Create an account at <https://render.com> and connect your GitHub.
2. **New → Blueprint** → pick `saurabh-03/aegis-auditor` → **Apply**.
   Render reads `render.yaml` and provisions everything, wiring `DATABASE_URL`,
   `REDIS_URL`, and a generated `JWT_SECRET` automatically.
3. When the **aegis-api** service is live, copy its URL
   (e.g. `https://aegis-api.onrender.com`) and set two env vars on **aegis-web**:
   - `API_ORIGIN = https://aegis-api.onrender.com`
   - `NEXT_PUBLIC_WS_ORIGIN = wss://aegis-api.onrender.com`
   then redeploy `aegis-web`.
4. Open the **aegis-web** URL — that's your public dashboard. 🎉

(The blueprint runs `prisma db push` before the API boots to create the schema.)

### Alternatives
- **Railway** — connect the repo, add Postgres + Redis plugins, deploy the
  `Dockerfile` as the API and a second service with start command `node dist/worker.js`,
  and deploy `web/` as a Node service. Same env vars as above.
- **Fly.io / any VPS with Docker** — use [`docker-compose.yml`](../docker-compose.yml)
  (API + worker + Postgres + Redis + web). Put it behind a TLS reverse proxy
  (Caddy/Traefik/nginx) and run `npm run db:push` once.

## Custom domain
On Render: **aegis-web → Settings → Custom Domain**, add e.g. `audit.yourdomain.com`,
and create the CNAME it shows at your DNS provider. TLS is issued automatically.
Point `API_ORIGIN`/`NEXT_PUBLIC_WS_ORIGIN` at a subdomain for the API if you want
(e.g. `api.yourdomain.com`).

## ⚠️ Safety, abuse & cost — read before going live

- **It scans other people's sites.** The passive scanner will fetch any public URL a
  visitor enters. That is normal for this category (like SSL Labs / Mozilla Observatory),
  but you are responsible for your instance's traffic. Keep rate limits low, and consider
  `REQUIRE_AUTH_FOR_SCAN=1` so only signed-in users can scan.
- **Active scans stay locked down.** Port scanning and sensitive-file probing require a
  *verified-ownership* project and never run for anonymous users — leave that gate in place.
- **SSRF is handled** — private/loopback/link-local/`169.254.169.254` targets are rejected.
  Do **not** set `ALLOW_PRIVATE_TARGETS` on a public instance. (Known limitation: entry-time
  DNS resolution doesn't fully defeat DNS-rebinding; for high-risk exposure, run the workers
  in a network-egress-restricted environment.)
- **Cost.** Free tiers sleep on idle and have limited resources — fine for a demo, not for
  real users. Budget for paid Postgres/Redis and always-on services before promoting it.
- **Secrets.** `JWT_SECRET` must be strong and private (the blueprint generates it). Rotate it
  if it ever leaks — all sessions invalidate.

## Production env summary

| Var | Set to | Why |
|-----|--------|-----|
| `DATABASE_URL` | Postgres URL | persistence (auto) |
| `REDIS_URL` | Redis URL | queue/workers (auto) |
| `JWT_SECRET` | strong random | session signing (auto-generated) |
| `AEGIS_ROLE` | `api` / `worker` | split enqueue vs. processing |
| `RATE_LIMIT_MAX` | e.g. `10` | abuse control |
| `REQUIRE_AUTH_FOR_SCAN` | `1` (optional) | gate the quick scan endpoint |
| `ALLOW_PRIVATE_TARGETS` | **unset** | keep SSRF protection on |
